**রিরাইট রুল:** শুধু এসএসএল সার্টিফিকেট ইনস্টল করলেই সব সমাধান হয়ে যায় না। ট্রাফিককে জোর করে সিকিউর লাইনে আনার জন্য `.htaccess` ফাইলিটিতে `RewriteRule ^(.*)$ https://%{HTTP_HOST}%{REQUEST_URI} [L,R=301]` কোডটি দেওয়া বাধ্যতামূলক।
</TLDR>

## "Not Secure" ওয়ার্নিং এর কারণ
সি-প্যানেলে AutoSSL ইনস্টল করার পর আপনি আনন্দে ব্রাউজারে `yourdomain.com` লিখলেন, কিন্তু গুগল ক্রোম আপনাকে বিশাল এক ওয়ার্নিং দিলো যে সাইটটি "Not Secure."

কেন এমন হলো? কারণ ইন্টারনেটের ডিফল্ট পোর্ট হলো 80 (HTTP)। আপনি যদি সার্ভারকে জোর করে পোর্ট 443 (HTTPS)-এ ট্রাফিক পাঠাতে নির্দেশ না দেন, ব্রাউজারগুলো ইনসিকিউর ভার্সনটি লোড করতেই থাকবে।
অ্যাচে (Apache) বা লাইটস্পিড (LiteSpeed) সার্ভারের ক্ষেত্রে আমরা `.htaccess` কনফিগারেশন ফাইল দিয়ে এর সমাধান করি।

### ধাপ ১: ওয়ার্ডপ্রেসের ভেতর সেটিংস পরিবর্তন
কোড ধরার আগে একদম বেসিক জিনিসটি ঠিক আছে কিনা যাচাই করুন:
১. আপনার WP-Admin ড্যাশবোর্ডে লগইন করুন।
২. **Settings > General**-এ যান।
৩. *WordPress Address (URL)* এবং *Site Address (URL)*—উভয় জায়গা থেকে `http://` রিমুভ করে `https://` লিখে দিন।
৪. Save Changes-এ ক্লিক করুন। (আপনাকে সাথে সাথে লগ আউট করে দেবে, ঘাবড়ানোর কিছু নেই)।

### ধাপ ২: .htaccess ফাইল এডিট করা
১. আমানা-ফ্লো সি-প্যানেল বা সাইবার-প্যানেলে (CyberPanel) ঢুকুন।
২. **File Manager** থেকে `public_html`-এ যান।
৩. ডানপাশের উপরের কোনায় "Settings" এ ক্লিক করে **Show Hidden Files (dotfiles)**-এ টিক দিন। (নইলে ফাইলিট দেখতে পাবেন না)।
৪. `.htaccess` ফাইলটির ওপর রাইট-ক্লিক করে **Edit** সিলেক্ট করুন।

### ধাপ ৩: Force HTTPS কোডটি বসানো
ফাইলের একদম *শুরুতে* (অর্থাৎ `# BEGIN WordPress` লেখাটির ওপরে) নিচের কোডটি কপি-পেস্ট করুন:

```apache
<IfModule mod_rewrite.c>
RewriteEngine On
RewriteCond %{HTTPS} off
RewriteRule ^(.*)$ https://%{HTTP_HOST}%{REQUEST_URI} [L,R=301]
</IfModule>
```
*এবার সেভ করে বেরিয়ে আসুন।*
এই কোডটি একটি স্থায়ী **301 রিডাইরেক্ট (301 Redirect)** তৈরি করে, যা গুগলকে এবং ব্রাউজারগুলোকে বলে দেয় যে, "আমার HTTP সাইটটি চিরকালের জন্য এনক্রিপ্টেড HTTPS-এ সরে গেছে।"

## Nginx সার্ভারে HTTPS রিডাইরেক্ট
আপনি যদি আনম্যানেজড উবুন্টু ভিপিএস-এ Nginx ব্যবহার করেন (যেখানে কোনো `.htaccess` থাকে না), তবে আপনাকে ইঞ্জিন-এক্স (Nginx) সার্ভার ব্লকে কোড দিতে হবে:

সার্ভার ব্লক ফাইলটি খুলে (যেমন: `/etc/nginx/sites-available/yourdomain.com`):
```nginx
server {
    listen 80;
    server_name yourdomain.com www.yourdomain.com;
    return 301 https://$host$request_uri;
}
```
এরপর `sudo systemctl reload nginx` কমান্ড দিলেই কাজ শেষ।

## FAQ
**প্রশ্ন: আমি জোর করে HTTPS রিডাইরেক্ট করার পর আমার সাইটটি "ERR_TOO_MANY_REDIRECTS" এরর দেখাচ্ছে। কেন?**  
উত্তর: এটি তখনই হয় যখন আপনি Cloudflare ব্যবহার করেন, এবং আপনার SSL/TLS সেটিংটি "Flexible"-এ রাখা থাকে। ক্লাউডফ্লেয়ার বারবার HTTP দিয়ে আপনার সার্ভারে ঢুকতে চায়, কিন্তু `.htaccess` তাকে জোর করে HTTPS-এ পাঠায়—এতে একটি ইনফিনিট লুপ তৈরি হয়। Cloudflare-এর ড্যাশবোর্ডে গিয়ে সেটিংটি পরিবর্তন করে **Full (Strict)** করে দিন, লুপ বন্ধ হয়ে যাবে।
