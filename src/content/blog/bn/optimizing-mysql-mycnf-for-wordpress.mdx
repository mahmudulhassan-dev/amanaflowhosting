<TLDR>
**মেমোরিই সব:** ডিফল্ট MySQL কনফিগারেশনে ক্যাশিং এর জন্য মাত্র ১২৮ এমবি (128MB) র‍্যাম ব্যবহার করে। আপনার সার্ভারে যদি ৮ জিবি র‍্যাম থাকে, তবে `innodb_buffer_pool_size` বাড়ানোর মাধ্যমে আপনি ডাটাবেসের স্পিড ১০ গুন পর্যন্ত বাড়াতে পারবেন।
</TLDR>

## ডিফল্ট MySQL এর ফাঁদ
যখন আমরা টার্মিনালে `apt-get` দিয়ে MySQL বা MariaDB ইনস্টল করি, তখন কনফিগারেশন ফাইলটি এমনভাবে সেট করা থাকে যেন এটি ১ জিবি র‍্যামের সস্তা সার্ভারেও চলতে পারে।

আপনি যদি অনেক দামি এবং শক্তিশালী সার্ভারে (যেমন ৮ জিবি র‍্যামের) একটি বড় ই-কমার্স সাইট রান করেন, MySQL নিজে থেকে সেই অতিরিক্ত র‍্যাম ব্যবহার করবে না। এটি সেই ১ জিবি র‍্যামের মতোই কাজ করবে। সার্ভার অপটিমাইজ করার দায়িত্ব আপনার।

### ধাপ ১: কনফিগ ফাইল ওপেন করা
SSH দিয়ে সার্ভারে লগইন করে ফাইলিট এডিটরের মাধ্যমে খুলুন:
```bash
sudo nano /etc/mysql/mysql.conf.d/mysqld.cnf
# অথবা MariaDB এর ক্ষেত্রে
sudo nano /etc/mysql/mariadb.conf.d/50-server.cnf
```

### ধাপ ২: InnoDB Buffer Pool কনফিগার 
InnoDB Buffer Pool হলো এমন একটি মেমরি এরিয়া যেখানে ডাটাবেসের টেবিল এবং ইনডেক্সগুলো ক্যাশ বা মেমোরিতে ধরে রাখা হয়। আপনার পুরো ডাটাবেস যদি বাফার পুলে জায়গা পেয়ে যায়, তবে কোয়েরিগুলো সরাসরি RAM থেকে রান করবে (যা অবিশ্বাস্য রকম ফাস্ট)।

স্ট্যান্ডার্ড নিয়ম হলো `innodb_buffer_pool_size`-কে **আপনার মোট সার্ভার র‍্যামের ৫০% থেকে ৬০%** করে দেওয়া। (বাকি র‍্যাম ওএস এবং পিএইচপি-এর জন্য ফাঁকা রাখতে হবে)।

```text
[mysqld]
innodb_buffer_pool_size = 4G
innodb_buffer_pool_instances = 4 
```
*(টিপস: প্রতি ১ জিবি বাফার পুলের জন্য একটি করে instance দিবেন)।*

### ধাপ ৩: কনকারেন্ট কানেকশন (Concurrent Connections)
ডিফল্টভাবে MySQL একসাথে মাত্র ১৫১ জনকে ডাটাবেস এক্সেস করার অনুমতি দেয়। ট্রাফিক স্পাইক হলে ১৫২ নম্বর ভিজিটর সাইটে ঢুকতে গেলেই ডাটাবেস এরর দেখতে পাবে।
```text
max_connections = 400
wait_timeout = 600
interactive_timeout = 600
```

### ধাপ ৪: কোয়েরি ক্যাশ (শুধুমাত্র MySQL 5.7 / MariaDB এর জন্য)
*ছোট নোট: MySQL ৮ ভার্সনে কোয়েরি ক্যাশ পুরো বাদ দিয়ে দেওয়া হয়েছে। আপনি যদি MariaDB ব্যবহার করেন, তাহলে এটি বেশ কাজের।*
```text
query_cache_type = 1
query_cache_limit = 2M
query_cache_size = 64M
```

সবশেষে ফাইলটি সেভ করে বের হয়ে ডাটাবেস ইঞ্জিন রিস্টার্ট দিন:
```bash
sudo systemctl restart mysql
```

## আপনার চেঞ্জগুলো ঠিকমতো কাজ করছে কি না বুঝবেন কীভাবে?
**MySQLTuner** নামের টুলটি নামিয়ে নিন। এটি একটি স্ক্রিপ্ট যা আপনার রানিং ডাটাবেস বিশ্লেষণ করে একদম সঠিক সাজেশন দেয়।

```bash
wget http://mysqltuner.pl/ -O mysqltuner.pl
chmod +x mysqltuner.pl
./mysqltuner.pl
```
তবে `my.cnf` পরিবর্তন করার সাথে সাথেই এটি রান করবেন না। সার্ভারে কিছু ট্রাফিক আসতে দিন। সাধারণত কনফিগার পরিবর্তনের ২৪ ঘন্টা পর এটি সবচেয়ে নিখুঁত রিপোর্ট দেয়।

## FAQ
**প্রশ্ন: আমি বাফার পুল বাড়ানোর সাথে সাথে সার্ভার ক্র্যাশ করছে কেন?**  
উত্তর: এর মানে আপনি আপনার সার্ভারে যতটুকু ফাঁকা র‍্যাম আছে তার চেয়ে বেশি ভ্যালু দিয়েছেন। টার্মিনালে `free -m` কমান্ড দিন। যদি ২ জিবি ফাঁকা র‍্যাম থাকে আর আপনি ৪ જીবি বাফার পুল দেন, তাহলে লিনেক্সের ওএস সাথে সাথে আপনার ডাটাবেস ফোর্সভালি স্টপ করে দেবে।
