**অটোমেটেড সিকিউরিটি:** Let's Encrypt সম্পূর্ণ বিনামূল্যে ৯০ দিন মেয়াদের এন্টারপ্রাইজ-গ্রেড এসএসএল প্রদান করে। Certbot টুলটি ব্যবহার করে কমান্ড লাইনের মাধ্যমে এই সার্টিফিকেটগুলো আনলিমিটেড সময়ের জন্য অটো-রিনিউ করা যায়।

## পেইড (Paid) এসএসএল এর দিন শেষ
১০ বছর আগে একটি এসএসএল সার্টিফিকেট কেনা বেশ ঝামেলা ও খরচের কাজ ছিল। বর্তমানে আপনি যদি ব্যাংক না হন তবে আপনার অবশ্যই **Let's Encrypt** এর ফ্রি এসএসএল ব্যবহার করা উচিত।

যদি আপনার আনম্যানেজড (মানুয়াল) সার্ভারে কোনো কন্ট্রোল প্যানেল না থাকে, তাহলে এটি দিয়ে সেকেন্ডের মধ্যেই Nginx বা Apache সার্ভার সুরক্ষিত করতে পারেন।

### ধাপ ১: Certbot ইনস্টল করা
Certbot টুলটি রান করার জন্য `snapd` প্যাকেজ ম্যানেজার লাগে, যা মডার্ন উবুন্টু ডিস্ট্রিবিউশনে ডিফল্টভাবেই থাকে।

সার্ভারে SSH দিয়ে কমান্ড দিন:
```bash
sudo snap install core; sudo snap refresh core
sudo snap install --classic certbot
sudo ln -s /snap/bin/certbot /usr/bin/certbot
```

### ধাপ ২: Nginx এর জন্য সার্টিফিকেট ইস্যু করা
Certbot এতোটাই স্মার্ট যে এটি নিজে থেকেই আপনার Nginx ফাইল রিড করে ডোমেইনের মালিকানা প্রমাণ করে এবং সাইটে এসএসএল বসিয়ে দেয়।

Nginx প্লাগইন রান করুন:
```bash
sudo certbot --nginx
```
Certbot আপনাকে কয়েকটি প্রশ্ন জিজ্ঞাসা করবে:
১. কোন ইমেইল ব্যবহার করবেন (রিনিউয়াল নোটিসের জন্য)।
২. কোন ডোমেইনের জন্য এসএসএল চান (এটি ডিফল্টভাবেই আপনার সাইটের নাম ডিটেক্ট করবে)।
৩. আপনি সব ট্রাফিককে HTTPS এ রিডাইরেক্ট করতে চান কিনা (অবশ্যই **Yes / Option 2** সিলেক্ট করবেন)।

*আপনি যদি Apache ব্যবহার করেন, তবে শুধু `sudo certbot --apache` রান করবেন।*

### ধাপ ৩: অটো-রিনিউয়াল চেক করা
Let's Encrypt এর সার্টিফিকেট প্রতি ৯০ দিন পর পর মেয়াদ হারায়। তবে Certbot দিনে দু'বার চেক করে অটোমেটিক্যালি মেয়াদ বাড়িয়ে দেয়।

প্রক্রিয়াটি ঠিকমতো কাজ করছে কিনা ড্রাই-রানের (dry-run) মাধ্যমে যাচাই করুন:
```bash
sudo certbot renew --dry-run
```
যদি এটি সাকসেসফুল দেখায়, তাহলে আপনাকে জীবনে আর কখনো ম্যানুয়ালি এসএসএল রিনিউ করতে হবে না।

## ডিএনএস (DNS) চ্যালেঞ্জের মাধ্যমে ওয়াইল্ডকার্ড সার্টিফিকেট
আপনার যদি সাবডোমেইন বা মাল্টিসাইট নেটওয়ার্কের জন্য ওয়াইল্ডকার্ড এসএসএল (`*.yourdomain.com`) লাগে, তখন Certbot সার্ভারে ফাইল বসিয়ে মালিকানা প্রমাণ করতে পারে না। তখন **DNS Challenge** ব্যবহার করতে হয়।

```bash
sudo certbot certonly --manual --preferred-challenges dns -d "*.yourdomain.com"
```
টুলটি আপনাকে একটি নির্দিষ্ট TXT ভ্যালু দেবে। আপনার ডোমেইনের কন্ট্রোল প্যানেলে গিয়ে একটি `_acme-challenge` TXT রেকর্ড তৈরি করুন এবং এটি প্রোপাগেট হওয়া পর্যন্ত অপেক্ষা করে এন্টার (Enter) চাপুন।

## FAQ
**প্রশ্ন: এসএসএল ইনস্টল হওয়ার পরও আমার ব্রাউজারে "Mixed Content" ওয়ার্নিং দেখাচ্ছে কেন?**  
উত্তর: আপনার সার্টিফিকেটে কোনো সমস্যা নেই! এর কারণ হলো আপনার সাইটের কোড অথবা ডাটাবেসে পুরোনো এইচটিটিপি (`http://`) লিংকে কোনো ছবি বা স্ক্রিপ্ট লোড করার চেষ্টা করা হচ্ছে। ডাটাবেসটি চেক করে ঐ লিংকগুলোকে `https://` এ আপডেট করে দিন।
