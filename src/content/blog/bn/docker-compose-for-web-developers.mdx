<TLDR>
**কন্টেইনারাইজেশন (Containerization):** ডকার আপনার কোডবেস, এর প্রয়োজনীয় ডিপেন্ডেন্সি এবং লিনেক্স এনভায়রনমেন্টকে একটি পোর্টেবল ইমেজে প্যাকেজ করে ফেলে। আর Docker Compose একাধিক ইমেজকে (যেমন অ্যাপ, ডাটাবেস এবং ক্যাশ) একসাথে রান করতে সাহায্য করে।
</TLDR>

## "আমার পিসিতে তো কাজ করছিল" সমস্যা
আপনি ম্যাকবুকে Node.js ২০ ব্যবহার করে একটি অ্যাপ বানালেন। কিন্তু ক্লায়েন্টের উবুন্টু সার্ভারে Node.js ১৪ ইনস্টল করা। কোড সেখানে ডিপ্লয় করতেই সাইট ক্র্যাশ করে এরর দেখানো শুরু করল।

এই জঘন্য সমস্যা সমাধানের জন্যই **Docker**-এর জন্ম।

### Docker Compose এর প্রয়োজনীয়তা কী?
একটি ফুল-স্ট্যাক Next.js অ্যাপ চালানোর জন্য ৩টি জিনিস লাগে:
১. Next.js ফ্রন্টএন্ড কন্টেইনার।
২. PostgreSQL ডাটাবেস কন্টেইনার।
৩. Redis ক্যাশ কন্টেইনার।

এই তিনটি জিনিস টার্মিনালে আলাদা আলাদা রান না করে, আমরা একটি `docker-compose.yml` ফাইলের সাহায্যে স্বয়ংক্রিয়ভাবে একসাথে রান করাতে পারি।

### ধাপ ১: সার্ভারে ডকার ইনস্টল করা
আমানা-ফ্লো এর একটি নতুন ক্লাউড সার্ভার নিয়ে থাকলে:
```bash
sudo apt update
sudo apt install docker.io docker-compose -y
sudo systemctl enable docker
sudo systemctl start docker
```

### ধাপ ২: `docker-compose.yml` ফাইল তৈরি
আপনার প্রজেক্টের মেইন ডিরেক্টরিতে ফাইলটি তৈরি করুন:
```yaml
version: '3.8'

services:
  web:
    image: node:20-alpine
    working_dir: /app
    volumes:
      - .:/app
    ports:
      - "3000:3000"
    command: npm run dev
    depends_on:
      - db
      - redis

  db:
    image: postgres:15-alpine
    environment:
      POSTGRES_USER: root
      POSTGRES_PASSWORD: strongpassword123
      POSTGRES_DB: myapp
    volumes:
      - pgdata:/var/lib/postgresql/data

  redis:
    image: redis:alpine

volumes:
  pgdata:
```

### ধাপ ৩: ম্যাজিক কমান্ড
সার্ভারে ডাটাবেস বা নোড আলাদাভাবে ইনস্টল করার কোনো দরকার নেই। টার্মিনালে শুধু লিখুন:
```bash
docker-compose up -d
```
ডকার নিজে থেকেই ডাটাবেস নামিয়ে নেবে, আপনার অ্যাপ বিল্ড করবে, পোর্ট 3000 ওপেন করবে এবং সবকিছু লাইভ করে দেবে!

## পারসিস্টেন্ট ভলিউম (Persistent Volumes)
YAML ফাইলটির সবচেয়ে গুরুত্বপূর্ণ অংশ হলো নিচের `volumes:` ব্লক। 
সাধারণত ডকার কন্টেইনার ডিলিট করলে ওর ভেতরের ডাটাবেসও মুছে যায়। কিন্তু ভলিউম ডিক্লেয়ার করার কারণে PostgreSQL এর ডাটা আপনার ভিপিএস-এর হার্ডড্রাইভে সেফ থাকবে। কন্টেইনার রিস্টার্ট দিলেও ডাটা হারাবে না।

## FAQ
**প্রশ্ন: আমি কি ডকারের ভেতরে সি-প্যানেল চালাতে পারব?**  
উত্তর: টেকনিক্যালি সম্ভব হলেও, এটা কোনো কাজের কথা নয়। সি-প্যানেল সরাসরি ওএস (OS) কার্নেলের কন্ট্রোল চায়, যা ডকারের দর্শনের বিরোধী। মডার্ন ডেভেলপমেন্টের জন্য CyberPanel বা CloudPanel ব্যবহার করুন।
