## পেজ ক্যাশ (Page Cache) এবং অবজেক্ট ক্যাশের পার্থক্য 
আমরা সবাই কোনো না কোনো ক্যাশিং প্লাগইন ব্যবহার করি। WP Rocket বা LiteSpeed Cache মূলত **Page Caching** করে—অর্থাৎ সাইটের তৈরি হওয়া এইচটিএমএল (HTML) সেভ করে রাখে।

কিন্তু পেজ ক্যাশ কাজ করে না যখন:
১. কোনো ইউজার ওয়েবাসাইটে লগ-ইন করে থাকেন।
২. উকমার্স (WooCommerce)-এ কাস্টমার চেকআউট করেন।
৩. কার্টে (Cart) কোনো আইটেম যোগ করা হয় (ডায়নামিক আপডেট)।

এমন পরিস্থিতিতে পেজ ক্যাশ বাইপাস হয়ে সরাসরি ডাটাবেসে রিকোয়েস্ট যায়। আর এই সমস্যা সমাধানেই কাজ করে **Object Caching (Redis)**।

### ধাপ ১: রেইডিস (Redis) সার্ভার ইনস্টল করা
আপনি যদি উবুন্টু ভিপিএস (Ubuntu VPS) ব্যবহার করেন, তবে টার্মিনাল থেকে ইনস্টল করুন:
```bash
sudo apt update
sudo apt install redis-server -y
```

রিবুট হলে যেন নিজে থেকেই চালু হয়:
```bash
sudo systemctl enable redis-server
sudo systemctl start redis-server
```

*(নোট: আমানা-ফ্লো-এর সি-প্যানেল (cPanel) বা ওয়ার্ডপ্রেস হোস্টিংয়ে Redis আগে থেকেই ১-ক্লিকে এনাবল করার সুবিধা থাকে)।*

### ধাপ ২: রেইডিস (Redis) মেমোরি ফিক্স করা
লিমিট না করে দিলে Redis আপনার ভিপিএস এর সম্পূর্ণ র‍্যাম দখল করে নিতে পারে। `/etc/redis/redis.conf` এডিট করে নিচের লাইনগুলো লিখুন:
```text
maxmemory 256mb
maxmemory-policy allkeys-lru
```

### ধাপ ৩: LiteSpeed Cache এর সাথে যুক্ত করা
আমরা ডিফল্ট "Redis Object Cache" প্লাগইনটির বদলে **LiteSpeed Cache (LSCache)** ব্যবহার করার পরামর্শ দিই।

১. ওয়ার্ডপ্রেস এডমিন প্যানেল থেকে **LiteSpeed Cache > Cache > Object**-এ যান।
২. **Object Cache** অফ থেকে `ON` করুন।
৩. Method সিলেক্ট করুন: `Redis`।
৪. Host: `127.0.0.1`
৫. Port: `6379`
৬. **Save Changes** এ ক্লিক করুন।

পেজ রিলোড দিলে দেখতে পাবেন "Connection Test: Passed" লেখা সবুজ কালারে দেখাচ্ছে।

## FAQ
**প্রশ্ন: আমি কি একটি Redis সার্ভার দিয়ে একাধিক সাইট চালাতে পারব?**  
উত্তর: হ্যাঁ, তবে আপনাকে অবশ্যই প্রতিটি সাইটের LiteSpeed Cache সেটিংসে আলাদা `Site ID` বা `Global Group` প্রিফিক্স (Prefix) দিতে হবে। নতুবা এক সাইটের ডাটা অন্য সাইটে চলে যাবে।
