## এ-পি-আই সিকিউরিটির গুরুত্ব
বর্তমানে অধিকাংশ বিজনেস মাইক্রো-সার্ভিস আর্কিটেকচার ব্যবহার করায় এ-পি-আই (API) হ্যাকারদের প্রধান লক্ষ্যবস্তুতে পরিণত হয়েছে। ডাটা চুরির একটি বড় অংশ এই এ-পি-আই থেকেই হয়ে থাকে।

### ১. অথেনটিকেশন (OAuth2/JWT)
সাধারণ পাসওয়ার্ড বা এ-পি-আই কি (API Key) ব্যবহার করা বন্ধ করুন। JWT বা JSON Web Token ব্যবহার করুন যার মেয়াদ হবে খুবই অল্প, যাতে টোকেন চুরি হলেও আপনার ডাটা বড় ক্ষতির মুখে না পড়ে।

### ২. রেট লিমিটিং (Throttling)
সুরক্ষাবিহীন এ-পি-আই যে কেউ একটি ছোট স্ক্রিপ্ট দিয়ে ডাউন করে দিতে পারে। আমানা-ফ্লো-এর লেয়ার-৭ মিটিগেশন ব্যবহার করে ইউজার নির্দিষ্ট সংখ্যক রিকোয়েস্টের বেশি যেন দিতে না পারে সেই ব্যবস্থা করুন।

### ৩. টি-এল-এস ১.৩ (TLS 1.3)
আপনার এ-পি-আই ট্রাফিক সবসময় এনক্রিপ্টেড রাখুন। ২০২৬ সালে টি-এল-এস ১.২ আর নিরাপদ নয়। সর্বোচ্চ নিরাপত্তার জন্য টি-এল-এস ১.৩ ভার্সন ব্যবহার করুন।

## এ-পি-আই সিকিউরিটি চেকলিস্ট
- [ ] সঠিক সি-ও-আর-এস (CORS) পলিসি চালু করুন।
- [ ] সেনসিটিভ ডাটার জন্য GET এর পরিবর্তে POST ব্যবহার করুন।
- [ ] সব ইনপুট ডাটা স্যানিটাইজ করুন (SQLi এবং XSS রুখতে)।
- [ ] প্রতিটি ফেইল্ড লগইন বা রিকোয়েস্টের লগ রাখুন।

## FAQ
**প্রশ্ন: আমি কি আমার এ-পি-আই এর সাথে ক্লাউড-ফ্লেয়ার ব্যবহার করতে পারবো?**  
উত্তর: জি, এটি খুবই ভালো আইডিয়া। এটি আপনার সার্ভারে ট্রাফিক পৌঁছানোর আগেই একটি বাড়তি সুরক্ষা দেয়।

**প্রশ্ন: গ্রাফ-কিউ-এল (GraphQL) কি REST এর চেয়ে বেশি নিরাপদ?**  
উত্তর: কোনটিই শতভাগ নিরাপদ নয়। গ্রাফ-কিউ-এল এর ক্ষেত্রে অতিরিক্ত "ডেপথ লিমিটিং" (Depth Limiting) খেয়াল রাখতে হয়।
