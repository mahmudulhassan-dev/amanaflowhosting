## `npm start` এর সমস্যা কোথায়?
নতুন ডেভেলপাররা যখন উবুন্টু ভিপিএস (Ubuntu VPS) কেনেন, তারা গিটহাব থেকে কোড ক্লোন করে `npm start` কমান্ড দিয়ে প্রজেক্ট রান করেন। 

অ্যাপটি চমৎকারভাবে চলতে থাকে—যতক্ষণ না ডেভেলপার তার SSH টার্মিনাল বন্ধ করেন। টার্মিনাল বন্ধ করার সাথে সাথেই অ্যাপটি ডাউন হয়ে যায় এবং ৫৬২ (502 Bad Gateway) এরর দেখায়।
এমনকি `nohup` ব্যবহার করলেও অ্যাপ ক্র্যাশ করলে এটি নিজে থেকে রিস্টার্ট হয় না।

### PM2 (Process Manager 2) এর কাজ কী?
PM2 হলো Node.js অ্যাপ্লিকেশন চালানোর জন্য একটি এন্টারপ্রাইজ-গ্রেড প্রসেস ম্যানেজার। এটি আপনার অ্যাপকে সবসময় লাইভ রাখে, ডাউনটাইম ছাড়াই রিলোড করতে পারে এবং লগ (log) ম্যানেজ করে।

### ধাপ ১: Node এবং PM2 ইনস্টল করা 
সার্ভারে লগইন করে Node.js এর স্টেবল ভার্সন ইনস্টল করুন:
```bash
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs
```
এবার PM2 গ্লোবালি ইনস্টল করুন:
```bash
sudo npm install pm2@latest -g
```

### ধাপ ২: অ্যাপ্লিকেশন রান করা
আপনার প্রজেক্ট ফোল্ডারে যান। 
যদি আপনার অ্যাপটি একটি **Express API** হয়:
```bash
pm2 start app.js --name "my-backend-api"
```

আর যদি অ্যাপটি বিল্ড (Build) করা **Next.js** প্রজেক্ট হয়:
```bash
pm2 start npm --name "nextjs-frontend" -- start
```

### ধাপ ৩: স্টার্টআপ (Startup) স্ক্রিপ্ট সেটআপ
সার্ভার রিবুট হলে PM2 যেন একা একাই পূর্বের অ্যাপগুলো রান করে, সেজন্য নিচের কমান্ডটি দিন:
```bash
pm2 startup
```
টার্মিনালে যে কোডটি আসবে সেটি কপি করে পেস্ট করুন। এরপর সেভ করুন:
```bash
pm2 save
```

## ধাপ ৪: রিভার্স প্রক্সি (Nginx)
আপনার Node অ্যাপটি এখন লোকালহোস্টে (যেমন: `localhost:3000`) চলছে। এটিকে ব্রাউজারের পোর্ট ৮০ (80) এর সাথে যুক্ত করতে Nginx কনফিগার করতে হবে। 

একটি Nginx কনফিগারেশন ফাইল তৈরি করুন:
```nginx
server {
    listen 80;
    server_name example.com;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```
Nginx রিস্টার্ট দিন এবং Certbot দিয়ে SSL এনাবল করুন। আপনার প্রোডাকশন সার্ভার এখন সম্পূর্ণ প্রস্তুত!

## FAQ
**প্রশ্ন: PM2 কি লোড ব্যালেন্সিং করতে পারে?**  
উত্তর: জি! আপনি যদি `pm2 start app.js -i max` রান করেন, তবে ভিপিএস-এর প্রতিটি সিপিইউ (CPU) কোরের জন্য একটি করে আলাদা Node প্রসেস তৈরি হবে, যা ইনকামিং ট্রাফিককে সমানভাবে ভাগ করে দেবে।
