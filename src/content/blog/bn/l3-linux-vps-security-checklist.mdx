## আনম্যানেজড সার্ভারের বাস্তবতা
আমানা-ফ্লো এর মতো কোনো প্রোভাইডার থেকে একটি লিনেক্স ক্লাউড ভিপিএস নিলে আপনি এর সম্পূর্ণ কন্ট্রোল (রুট অ্যাক্সেস) পেয়ে যান। কিন্তু এর সাথে সাথে সার্ভার সুরক্ষিত রাখার দায়িত্বও আপনার কাঁধে চলে আসে।

আমাদের এল-৩ (L3) সিস্টেম অ্যাডমিনরা সার্ভার সুরক্ষিত করতে যে চেকলিস্ট অনুসরণ করেন তা নিচে দেওয়া হলো।

### ১. SSH হার্ডেনিং (Hardening)
ডিফল্ট এসএসএইচ (SSH) কনফিগারেশন খুবই ঝুঁকিপূর্ণ। `/etc/ssh/sshd_config` ফাইলটি ওপেন করে এই রুলগুলো অ্যাপ্লাই করুন:
- **রুট লগইন বন্ধ করুন:** `PermitRootLogin no`
- **শুধু কি-বেসড অথ (Key-based Auth):** `PasswordAuthentication no`
- **পোর্ট পরিবর্তন করুন:** `Port 2222` (অথবা অন্য যেকোনো পোর্ট)

*এটি অ্যাপ্লাই করার পর SSH ডেমন (daemon) রিস্টার্ট দিন।*

### ২. UFW ফায়ারওয়াল চালু করুন
কখনোই অপ্রয়োজনীয় কোনো পোর্ট ওপেন রাখবেন না।
```bash
sudo ufw default deny incoming
sudo ufw default allow outgoing
sudo ufw allow 2222/tcp # আপনার নতুন SSH পোর্ট
sudo ufw allow 80/tcp   # HTTP
sudo ufw allow 443/tcp  # HTTPS
sudo ufw enable
```

### ৩. Fail2Ban ডিপ্লয়মেন্ট
যেসব বট আপনার সার্ভারের পাসওয়ার্ড গেস করার চেষ্টা করে, Fail2Ban তাদের লগ পড়ে স্বয়ংক্রিয়ভাবে ঐ আইপি (IP) গুলো ব্যান করে দেয়।

### ৪. অটোমেটিক আপডেট
উবুন্টু (Ubuntu) বা ডেবিয়ান (Debian)-এ `unattended-upgrades` কনফিগার করুন যাতে কোনোরকম হস্তক্ষেপ ছাড়াই সার্ভার নিজে নিজেই সিকিউরিটি প্যাচ আপডেট করে নিতে পারে।

### ৫. ম্যালওয়্যার স্ক্যানিং
আপনি যদি ক্লায়েন্টদের সাইট হোস্ট করেন, তারা থার্ড-পার্টি থেকে নানা রকম আক্রান্ত প্লাগইন আপলোড করবেই। তাই নিয়মিত ClamAV অথবা Linux Malware Detect (Maldet) দিয়ে স্ক্যান করানো জরুরি।

## FAQ
**প্রশ্ন: আমি যদি সার্ভার থেকে লক-আউট হয়ে যাই, তাহলে কী হবে?**  
উত্তর: চিন্তার কিছু নেই। আমানা-ফ্লো ক্লায়েন্ট এরিয়া থেকে ইমার্জেন্সি VNC কনসোলের মাধ্যমে আপনি সবসময় সার্ভারে ঢুকতে পারবেন।
